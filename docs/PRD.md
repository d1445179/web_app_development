# 待辦事項管理系統 PRD (Product Requirements Document)

## 1. 專案概述
- **背景與動機**：在快節奏的生活與學習中，學生與一般使用者常會忘記重要的待辦事項。一個簡單易用的待辦事項系統能幫助他們追蹤並管理每日任務，確保不會遺漏重要事項，進而提升個人學習與生活效率。
- **目標用戶**：學生 / 一般使用者
- **核心價值主張**：提供直覺、輕量且無負擔的任務管理介面，讓使用者能輕鬆記錄、追蹤並完成每日目標。

## 2. 功能需求

以下為系統主要功能及其對應的使用者故事（User Stories）：

1. **新增任務**
   - 作為 **[學生 / 一般使用者]**，我希望 **[能輸入任務的標題與內容並新增]**，以便 **[隨時記錄需要完成的待辦事項]**。

2. **任務列表顯示**
   - 作為 **[學生 / 一般使用者]**，我希望 **[能在畫面上看見所有已新增的任務列表]**，以便 **[清楚掌握目前有哪些事情尚待處理]**。

3. **編輯任務**
   - 作為 **[學生 / 一般使用者]**，我希望 **[能在資料輸入錯誤或需要補充細節時修改任務內容]**，以便 **[維持任務資訊的正確性與即時性]**。

4. **刪除任務**
   - 作為 **[學生 / 一般使用者]**，我希望 **[能刪除不再需要執行或輸入錯誤的任務]**，以便 **[保持任務列表的整潔與精確]**。

5. **標記完成**
   - 作為 **[學生 / 一般使用者]**，我希望 **[能將已做完的任務標明為完成狀態]**，以便 **[獲得成就感並輕鬆辨識尚未完成的項目]**。

## 3. 非功能需求
- **技術限制**：預計採用 Flask + Jinja2 + SQLite 作為核心技術堆疊。
- **效能考量**：操作響應時間需快速（畫面無延遲感或等待時間不大於 1 秒），介面在網頁和行動裝置上應有基礎的 RWD 佈局以利一般使用者觀看。
- **安全考量**：需防範基本的 XSS 或 SQL Injection 攻擊（如藉由 ORM 或參數化查詢來存取資料庫）。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須具備)**：
  - 首頁能顯示所有任務列表
  - 可新增基本文字任務
  - 可刪除任務
  - 可切換任務為「已完成」狀態
- **Should Have (應該具備)**：
  - 編輯任務功能
  - 介面區分「進行中」與「已完成」的任務清單
- **Nice to Have (有更好)**：
  - 任務分類功能（工作、學習、私人等）
  - 截止日期及倒數計時功能
  - 會員系統（多帳號支援）
